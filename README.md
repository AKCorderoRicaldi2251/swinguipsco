# Dice Shooter

This project is a collaborative Java application developed for the **Professional and Study Competencies for Computing (PSCO)** module. It features a collection of four dice-based games with a SwingUI developed.
---

##  The Games
The application offers four distinct mini-games accessible through a centralized menu interface.

### 1. Dice Patterns Challenge
* **Goal**: Match specific dice patterns (similar to poker hands) to score points.
* **Mechanics**: Roll five dice with up to two rerolls allowed per round.
* **Scoring**: Patterns include Five of a Kind (50 pts), Full House (35 pts), and Straights (30 pts).
* **Interaction**: Shoot individual dice to "Lock" them before hitting the **REROLL** target.

### 2. Dice Grid Puzzle
* **Goal**: Fill a 3x3 grid with dice rolls to form high-scoring patterns in every row and column.
* **Mechanics**: Place one die at a time until all 9 cells are filled.
* **Scoring**: Points are awarded for Three of a Kind (15 pts), Straights (12 pts), and Pairs (8 pts).
* **Interaction**: Shoot an empty grid slot to place the current "Data Die".

### 3. Dice Codebreaker
* **Goal**: Deduce a secret 4-dice code (values 1–6) generated by the computer.
* **Feedback System**: After each submission, the UI provides color-coded feedback:
    * **Green Border**: Correct number in the correct position (Hit).
    * **Yellow Border**: Correct number in the wrong position (Blow).
* **Constraint**: The player must breach the code within 10 attempts.

### 4. System Crash (Custom Game)
* **Goal**: Make the computer reach 0 dice before you by betting or calling liar 
* **Rules**: Use computer's bets to gauge what dice they have. If someone bets 5 4s, you can assume they probably have some 4s in their hand. Combine that with the knowledge of your own dice and you can make a pretty good guess at how many are on the table
* **Stats**: Tracks successful breaches (**Wins**) versus failures (**Losses**).

## Core System Architecture

### **1. Scene Management (`SceneManager.java`)**
The `SceneManager` acts as the central controller for the application.
* **Centralized Input**: It manages all mouse motion and click events, ensuring consistent crosshair and bullet firing across every scene.
* **Smooth Transitions**: Implements professional alpha-composite fade-out and fade-in effects when switching between game modules.
* **Main Game Loop**: Runs a centralized timer to trigger logic updates and repaints.

### **2. The "Shoot-to-Input" Framework (`ShootMenu.java`)**
All game UIs inherit from the `ShootMenu` abstract class, defining the unique interaction style.
* **Target System**: Interactive elements are `Target` objects with hitboxes. Players must "shoot" these targets with bullets to trigger selections.
* **Dynamic Sway**: Features a camera drift effect (`camX`/`camY`) that shifts the UI slightly based on mouse movement to create depth.
* **Bullet Physics**: Handles the lifecycle of `Bullet` objects, checking for intersections with target hitboxes in real-time.

### **3. Data Persistence (`GameDataManager.java`)**
* **JSON Storage**: Player profiles and scores are serialized into `.json` files within a `/saves/` directory.
* **Global Rankings**: A terminal-style leaderboard displays high scores and Win/Loss ratios for all registered hackers.
